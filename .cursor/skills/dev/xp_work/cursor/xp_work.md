# XP Work Manager Skill

Follow **`SKILL.md`** in this same directory (for example `.cursor/skills/dev/xp_work/SKILL.md` when the skill is installed). It is the single source of truth for:

- Migrating legacy `work/FEEDBACK.md` → `work/IMPROVEMENTS.md`
- Bootstrapping all canonical `work/*.md` files with purpose headers so they can be committed early
- Triggers: `improve:`, `feedback:`, `deferred:`, `tech debt:` / `debt:`, `feat:` / `todo:`, `bug:`, `goal:`, iteration and acceptance commands
- Templates under `assets/`

Do not implement product work—only record and manage tracking files as described in `SKILL.md`.
